# 重放大师 Mod — 完整编写流程

本文档串联从环境到可玩版本的步骤，并与社区教程对照。

## 1. 环境

- [.NET 9 SDK](https://dotnet.microsoft.com/download)
- [Godot 4.5.1 .NET（Mono）](https://godotengine.org/download/archive/4.5.1-stable/) — 版本需与游戏一致，否则 pck 可能无法加载
- Steam 版《杀戮尖塔 2》
- 教程参考：[环境配置](https://glitchedreme.github.io/SlayTheSpire2ModdingTutorials/)（站点首页导航「基础 → 01 环境配置」）

## 2. 工程结构（本仓库）

- `ReplayMaster.csproj`：引用 `sts2.dll`、`0Harmony.dll`、NuGet **`Alchyr.Sts2.BaseLib` 0.2.0**、ModAnalyzers、Publicizer；构建后复制到 `mods/ReplayMaster` 与 `mods/BaseLib`
- **`mod_manifest.json`**：Mod 清单（游戏识别此文件名），`dependencies` 含 `BaseLib`
- `Scripts/`：`Entry.cs`（`ModInitializer` + Harmony）、`ReplayMasterCard.cs`、`StartingDeckPatch.cs`
- `ReplayMaster/`：卡图、`mod_image.png`、本地化 `localization/zhs/*.json`
- `export_presets.cfg`：Godot 导出 `BasicExport`

## 3. 编写卡牌逻辑（BaseLib）

- 继承 `CustomCardModel`，`[Pool(typeof(ColorlessCardPool))]`
- 固有：`CanonicalKeywords` 含 `CardKeyword.Innate`
- 重放层数：与原版 `HiddenGem` 一致，使用 `IntVar("Replay", …)`、`ExtraHoverTips` 使用 `StaticHoverTip.ReplayStatic`，打出后对目标牌 `BaseReplayCount += …` 并 `CardCmd.Preview`
- 手牌选择：`CardSelectCmd.FromHand`，`filter` 排除当前打出的 `cardPlay.Card`
- 教程：[BaseLib 接口](https://glitchedreme.github.io/SlayTheSpire2ModdingTutorials/)

## 4. 开局卡组 +1（Harmony）

- 对 `Player.PopulateStartingDeck` 使用 **Postfix**：在初始卡组生成后 `Deck.AddInternal` 一张 `ReplayMasterCard` 的可变克隆，并按 `Id` 去重

## 5. 本地化

- `ReplayMaster/localization/zhs/cards.json`：卡牌标题与描述（占位符 `{Replay:diff()}`）
- `replay_master_ui.json`：选牌提示 `HAND_SELECT_REPLAY`

## 6. 构建与安装

```powershell
cd replay-master
dotnet build -c Debug
```

检查：

- `Slay the Spire 2/mods/ReplayMaster/ReplayMaster.dll`
- `Slay the Spire 2/mods/ReplayMaster/mod_manifest.json`
- `Slay the Spire 2/mods/BaseLib/*`（三文件）

## 7. 导出 pck

1. Godot 打开本工程目录。
2. **项目 → 导出 → BasicExport → 导出 PCK**，保存为 `mods/ReplayMaster/ReplayMaster.pck`。
